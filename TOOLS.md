# Tools and Resources

Stuff that actually helps with investigations. I've tested or verified most of these, but use your own judgment.

---

## Investigation Tools

### OSINT (Open Source Intelligence)

**SpiderFoot** - https://github.com/smicallef/spiderfoot
- Automated OSINT gathering
- Lots of integrations (Shodan, VirusTotal, etc.)
- Free version has limits, paid version is expensive

**Maltego** - https://www.maltego.com/
- Visual link analysis tool
- Good for mapping networks and relationships
- Commercial, but has community edition

**theHarvester** - https://github.com/laramies/theHarvester
- Email, subdomain, name harvesting
- Useful for initial reconnaissance

### Network Analysis

**Wireshark** - https://www.wireshark.org/
- Network protocol analyzer
- Essential for traffic analysis
- Steep learning curve but powerful

**Nmap** - https://nmap.org/
- Network discovery and security auditing
- Standard tool for network mapping

**Metasploit** - https://www.metasploit.com/
- Penetration testing framework
- Extensive exploit database

### Evidence Collection

**OBS Studio** - https://obsproject.com/
- Screen recording and streaming
- Good for documenting online evidence
- Free and open source

**HashMyFiles** - https://github.com/FiloSottile/HashMyFiles
- File hashing and verification
- Create evidence hashes for integrity checks

## Security Tools

### Encryption

**VeraCrypt** - https://www.veracrypt.fr/
- Whole disk encryption
- Creates encrypted containers
- Open source, audited

**Signal Desktop** - https://signal.org/
- Encrypted messaging
- Considered secure by most experts

**ProtonMail** - https://proton.me/
- Encrypted email service
- Based in Switzerland (privacy laws)

### Anonymity

**Tor Browser** - https://www.torproject.org/
- Anonymous web browsing
- Built-in privacy protections
- Slow but secure

**Mullvad VPN** - https://mullvad.net/
- Privacy-focused VPN
- No-logs policy
- Based in Panama

### System Security

**Linux Hardening** - https://github.com/konstruktoid/Audit-and-Hardening
- Security configurations for Linux
- Reduces attack surface

**Windows Privacy** - https://www.privacytools.io/
- Windows privacy and security tools
- Telemetry blocking, etc.

## Research Tools

### Documentation

**Obsidian** - https://obsidian.md/
- Knowledge management
- Good for linking evidence and theories
- Local first, encrypted sync available

**Logseq** - https://logseq.com/
- Alternative to Obsidian
- Graph-based knowledge management
- Good for visualizing connections

### Academic Research

**Google Scholar** - https://scholar.google.com/
- Academic paper search
- Essential for finding related research
- Advanced search operators available

**IEEE Xplore** - https://ieeexplore.ieee.org/
- Technical paper database
- Good for cybersecurity research
- Some papers require subscription

## Specialized Tools

### Supply Chain Security

**Dependency Check** - https://dependencycheck.org/
- Scans dependencies for vulnerabilities
- Works with multiple package managers
- Free for open source

**Snyk** - https://snyk.io/
- Commercial vulnerability scanning
- CI/CD integration available
- Good for development teams

### Malware Analysis

**Cuckoo Sandbox** - https://cucko.sh/
- Automated malware analysis
- Safe environment for suspicious files
- Complex setup but powerful

**YARA** - https://virustotal.github.io/yara/
- Malware identification and classification
- Rule-based matching system
- Industry standard

## Communication Tools

### Secure Messaging

**Wire** - https://wire.com/
- Secure messaging app
- End-to-end encryption
- Open source

**Session** - https://getsession.org/
- Privacy-focused messaging
- Minimal metadata collection
- Onion routing available

### Voice/Video

**Jitsi Meet** - https://jitsi.org/
- Self-hosted video conferencing
- End-to-end encryption available
- No account required

## Forensics

**Autopsy** - https://www.autopsy.com/
- Digital forensics platform
- Good for disk analysis
- Free for law enforcement

**Volatility** - https://www.volatilityfoundation.org/
- Memory forensics
- Extracts artifacts from RAM dumps
- Advanced techniques

## Warning: Tool Selection

**Free vs Paid:**
- Free tools often have limitations
- Paid tools usually more powerful
- Consider your budget and needs

**Legality:**
- Some tools may be illegal in your jurisdiction
- Understand local laws before using
- Get proper authorization when required

**Trust:**
- Open source tools can be audited
- Commercial tools may have backdoors
- Verify tool authenticity before use

## Getting Started

1. **Start with OSINT** - Gather public information first
2. **Use analysis tools** - Make sense of the data
3. **Document everything** - Create evidence logs
4. **Stay secure** - Protect your own systems
5. **Verify findings** - Don't trust single sources
